# 家庭健康小程序页面原型总说明

## 1. 原型目标

本原型用于指导后续微信小程序页面设计与开发。产品定位为：体检报告 / 化验单解读、家庭健康档案管理、重点指标趋势追踪、复查与生活提醒。

本原型不做营销页，不做 AI 医生，不做诊断、处方或治疗方案。

核心体验：

```text
首次进入
→ 上传我的体检报告
→ AI 解读报告
→ 用户确认关键指标
→ 保存到健康档案
→ 首页展示今日重点
→ 成员详情持续追踪趋势与提醒
```

## 2. 设计方向

```text
视觉方向：温润健康管家
视觉记忆点：家人档案卡
使用场景：报告解读 + 日常管理
硬约束：中老年友好、少素材依赖
```

首页、家人页、成员详情页都应服务应用工具界面 (utility surface)：让用户定位状态、看到重点、完成下一步行动。

## 3. 全局导航

底部 Tab 固定为：

```text
首页 / 家人 / 报告 / 我的
```

> 2026-06-18 经 OpenDesign 设计稿确认，由 3 Tab（首页/家人/更多）调整为 4 Tab（首页/家人/报告/我的），新增「报告」一级入口承载跨家人的报告列表，「更多」改名为「我的」。

### 3.1 首页

职责：家庭健康总览。

回答：

1. 谁需要关注？
2. 哪个指标在变化？
3. 下一步该做什么？

### 3.2 家人

职责：家庭成员健康档案入口。

原则：

- 无档案时，引导创建「我」的档案。
- 有档案时，优先展示已有家人。
- 添加家人入口存在，但不抢主视觉。

### 3.3 报告

职责：跨家人的体检报告 / 化验单列表入口。

承载：所有成员的报告列表、按状态筛选、上传新报告、进入报告解读。

### 3.4 我的

职责：低频设置与合规说明。

承载：提醒管理、数据导出、AI 设置、隐私说明、免责声明、关于产品。

## 4. 主流程

```text
[首页首次使用]
  ├─ 上传我的体检报告
  │   ├─ 选择 / 默认成员：我
  │   ├─ 上传图片或 PDF
  │   ├─ AI 调用提示
  │   ├─ 解读中
  │   ├─ 报告解读结果
  │   ├─ 指标确认
  │   └─ 保存到成员档案
  │       ├─ 回到成员详情
  │       └─ 首页刷新今日重点
  │
  ├─ 完善我的基础信息
  │   └─ 成员编辑页
  │
  └─ 添加家人
      └─ 添加 / 编辑家人页
```

## 5. 页面清单

| 页面 | 类型 | MVP | 说明 |
|---|---|---:|---|
| 首页 | Tab | 是 | 家庭健康总览、今日重点、提醒、最近报告 |
| 家人页 | Tab | 是 | 家人档案卡列表、首次使用引导 |
| 报告页 | Tab | 是 | 跨家人报告列表、按状态筛选、上传新报告 |
| 我的页 | Tab | 是 | 低频设置、隐私、免责声明 |
| 成员详情页 | 详情 | 是 | 单个成员的健康档案中心 |
| 添加 / 编辑家人页 | 表单 | 是 | 创建和维护成员基础信息 |
| 报告上传页 | 流程 | 是 | 上传报告并明确成员归属 |
| 解读中页 | 状态 | 是 | AI 解读等待与失败兜底 |
| 报告解读结果页 | 结果 | 是 | 固定结构输出 AI 解读 |
| 指标确认页 | 表单 | 是 | 用户确认后写入指标库 |
| 指标趋势详情页 | 详情 | Phase 3 | 单指标长期趋势 |
| 提醒列表页 | 列表 | Phase 3 | 复查、复诊、习惯提醒管理 |
| 提醒编辑页 | 表单 | Phase 3 | 创建和修改提醒 |


### 6.2 首次使用状态

```text
┌────────────────────────┐
│ 先建立你的健康档案        │
│ 上传一份体检报告，先看懂重点 │
│ [上传我的体检报告]        │

└────────────────────────┘

┌────────────────────────┐
│ 也可以管理家人健康        │

```

### 6.3 首页规则

- 首页最多显示 5 个主要模块。
- 异常状态使用柔和橙红，不使用大面积红色。
- 提醒排序：逾期 > 今日 > 未来 7 天 > 普通习惯。
- 今日重点必须指向一个具体行动。

## 10. 报告上传页原型

```text
上传报告

归属成员
[我 ▾]

报告文件
┌────────────────────────┐
│ 上传体检报告或化验单       │
│ 支持图片 / PDF            │
│ [拍照上传] [选择文件]      │
└────────────────────────┘

报告信息
报告类型  [体检 / 化验 / 门诊 / 影像]
报告日期  [选择日期]
医院 / 科室 [可选]

AI 解读提示
报告解读需要调用第三方 AI 服务，报告文本可能会被发送至模型接口用于生成解读。

[开始解读]
[先保存，不解读]
```

规则：

- 默认归属成员为「我」。
- 上传失败不丢失已填写信息。
- AI 调用提示必须在开始解读前展示。

## 11. 解读中页原型

```text
正在整理报告重点

○ 读取报告内容
○ 整理异常指标
○ 生成通俗解释
○ 准备确认指标

这不是诊断结果，只用于帮助理解报告。

[稍后查看]
```

规则：

- 文案不使用「诊断中」。
- 解读超时后允许稍后查看。
- 解读失败后保留报告文件，可重新解读或手动录入指标。

## 12. 报告解读结果页原型

```text
报告解读
2026-06-01｜我的体检报告

┌────────────────────────┐
│ 报告摘要                 │
│ 本次报告有 3 项指标需要关注 │
└────────────────────────┘

┌────────────────────────┐
│ 异常指标                 │
│ 尿酸 468 μmol/L｜偏高     │
│ 可能与高嘌呤饮食、饮酒、代谢因素相关 │
│ 建议：减少酒精和高嘌呤食物，按计划复查 │
└────────────────────────┘

┌────────────────────────┐
│ 生活方式建议              │
│ 饮食：减少动物内脏、海鲜和酒精 │
│ 运动：保持规律中等强度运动   │
└────────────────────────┘

┌────────────────────────┐
│ 复查 / 就医提醒           │
│ 建议 1-3 个月后复查尿酸    │
│ 如有不适或持续异常，请咨询医生 │
└────────────────────────┘

免责声明：内容仅用于帮助理解体检报告和管理健康档案，不构成诊断或治疗建议。如指标持续异常或出现不适，请咨询医生。

[确认关键指标并保存]
```

规则：

- 固定结构输出：摘要、异常指标、重点关注、可能影响因素、生活方式建议、复查 / 就医提醒、免责声明。
- 不给诊断结论、处方、药物剂量。
- 异常指标用「指标批注卡」展示。

## 13. 指标确认页原型

```text
确认关键指标
2026-06-01｜我的体检报告

┌────────────────────────┐
│ 尿酸                    │
│ 数值：468                │
│ 单位：μmol/L             │
│ 参考范围：男 149-416      │
│ 状态：偏高                │
│ [编辑] [忽略]             │
└────────────────────────┘

┌────────────────────────┐
│ LDL-C                   │
│ 数值：3.72               │
│ 单位：mmol/L             │
│ 状态：偏高                │
│ [编辑] [忽略]             │
└────────────────────────┘

[添加指标]
[保存到健康档案]
```

规则：

- 未确认指标不能进入趋势统计。
- 允许编辑、忽略、添加指标。
- 保存失败必须保留已编辑内容。

## 18. 原型验收

- 能从首次使用走完上传报告、AI 解读、确认指标、保存档案的闭环。
- 首页、家人页、成员详情页不是功能堆叠，而是状态导向。
- 家人档案卡是最明确的视觉记忆点。
- AI 解读始终有边界提示和免责声明。
- 未确认指标不进入趋势统计。
- 所有关键页面都有下一步行动。
