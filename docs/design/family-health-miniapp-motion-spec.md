# 家庭健康小程序动效规范

## 1. 动效目标

动效用于帮助用户理解层级、状态变化和操作反馈，不用于炫技。健康类产品需要低压力、稳定、可信的体验，动效必须克制。

已确认三个代表性微交互都保留：

1. 档案卡浮入。
2. 指标批注点亮。
3. 状态环展开。

优先级：

```text
档案卡浮入 > 指标批注点亮 > 状态环展开
```

## 2. 全局动效规则

### 2.1 允许的动画属性

只使用：

```text
transform
opacity
```

谨慎使用：

```text
background-color
```

仅用于短暂状态过渡，不能循环闪烁。

### 2.2 禁止

- 禁止 `transition: all`。
- 禁止动画 width、height、margin、padding。
- 禁止弹跳、旋转、持续呼吸灯。
- 禁止大面积颜色闪烁。
- 禁止用动效表达医疗严重程度。
- 禁止让关键信息必须依赖动画才能被理解。

### 2.3 推荐时长

| 类型 | 时长 |
|---|---:|
| 点击反馈 | 80-120ms |
| 卡片进入 | 180-240ms |
| 状态高亮 | 160-220ms |
| 展开 / 收起 | 220-280ms |
| 页面级轻转场 | 240-320ms |

### 2.4 缓动

推荐使用平滑减速：

```text
cubic-bezier(0.16, 1, 0.3, 1)
```

避免弹性 bounce。

## 3. 微交互一：档案卡浮入

### 3.1 优先级

P0，最高优先级。

原因：家人档案卡是产品视觉记忆点，浮入能强化「家庭档案」的陪伴感。

### 3.2 使用场景

- 家人页首次加载已有档案。
- 新增家人后返回家人列表。
- 首页今日重点卡出现。
- 首次使用引导卡出现。

### 3.3 动效表现

```text
初始：opacity 0, translateY(12px)
结束：opacity 1, translateY(0)
时长：200ms
延迟：多卡片错峰 40ms
```

### 3.4 边界

- 同屏最多前 3 张卡片错峰浮入。
- 不使用缩放过大、旋转、弹跳。
- 列表很长时，只对首屏可见卡片做一次进入。
- 下拉刷新后不重复大规模动画。

### 3.5 reduced-motion 降级

- 直接显示卡片。
- 可保留极轻 opacity 变化，或完全无动画。

## 4. 微交互二：指标批注点亮

### 4.1 优先级

P0 / P1，第二优先级。

原因：它直接服务报告解读，让用户知道「哪里需要关注」。

### 4.2 使用场景

- 报告解读结果页首次进入，逐条强调异常指标。
- 指标确认页保存成功后，已确认指标短暂反馈。
- 趋势图点击异常点后，对应指标批注卡点亮。

### 4.3 动效表现

推荐组合：

```text
opacity: 0.86 → 1
translateY(6px) → translateY(0)
背景浅色从透明 → 状态浅底色
时长：180-220ms
```

### 4.4 点亮规则

- 同屏最多点亮 3 个指标。
- 如果异常项很多，只点亮前 3 个最重要指标。
- 点亮是一次性反馈，不循环。
- 不用红色闪烁。
- 不用呼吸灯。

### 4.5 状态色

- 偏高 / 偏低：柔和橙色浅底。
- 明确异常：小面积橙红标签。
- 待确认：蓝色浅底。

### 4.6 reduced-motion 降级

- 直接显示静态高亮状态。
- 用左侧短状态条或状态标签表达选中。

## 5. 微交互三：状态环展开

### 5.1 优先级

P1，第三优先级。

原因：状态环有品牌感，但容易过度设计或误导成健康评分，因此必须谨慎。

### 5.2 使用场景

- 成员详情页点击健康状态环。
- 从首页今日重点进入成员详情时，可展示展开态。
- 不建议在首页自动展开。

### 5.3 动效表现

```text
状态环：scale(0.98) → scale(1)
明细：opacity 0 → 1, translateY(8px) → 0
时长：240-280ms
```

### 5.4 展开内容

状态环展开后只展示事实聚合：

```text
2 项需关注
1 个复查提醒
1 个指标连续升高
```

不展示：

```text
健康分
健康等级
疾病风险概率
```

### 5.5 边界

- 不做环形进度持续旋转。
- 不做分数滚动。
- 不用复杂 Canvas 动画。
- 状态环不承担诊断表达。

### 5.6 reduced-motion 降级

- 点击后直接切换展开 / 收起。
- 保留文本明细。

## 6. 点击反馈

### 6.1 按钮

主按钮、次按钮可使用：

```text
active: scale(0.96)
时长：80-120ms
```

规则：

- 只作用于可点击按钮，不作用于整页。
- 不影响文本可读性。
- 不做弹跳。

### 6.2 卡片

家人档案卡点击时可使用：

```text
active: scale(0.985)
opacity: 0.96
```

不建议大幅缩放，避免卡片文字抖动。

## 7. 加载与处理中

### 7.1 骨架屏

用于：

- 首页总览加载。
- 家人列表加载。
- 成员详情加载。

规则：

- 保持页面结构稳定。
- 不使用大面积 spinner。
- 动效可静态或低频 shimmer；低端设备可关闭。

### 7.2 解读中

用于 AI 报告解读。

表现：

```text
读取报告内容
→ 整理异常指标
→ 生成通俗解释
→ 准备确认指标
```

规则：

- 文案不出现「诊断中」。
- 可使用轻量 Lottie，但不是必须。
- Lottie 只用于等待和完成反馈，不进入核心布局。

### 7.3 上传中

表现：

- 文件名。
- 上传进度。
- 取消入口。

规则：

- 不跳转页面。
- 不清空已填信息。

## 8. 页面切换

微信小程序原生页面切换能力有限，不做复杂页面过渡。

建议：

- 页面进入后核心卡片轻微浮入。
- 不自定义沉重路由动画。
- 不用大量 Lottie 作为页面过渡。

## 9. Lottie 使用边界

### 9.1 可用场景

- AI 解读中。
- 上传完成反馈。
- 首次使用空态的小型温和动效。

### 9.2 禁止场景

- 每张家人卡都使用 Lottie。
- 趋势图用 Lottie 代替真实图表。
- 异常指标用 Lottie 警报。
- 页面背景装饰性 Lottie。

### 9.3 加载策略

- Phase 1 可不引入 Lottie。
- Phase 2 接入 AI 解读后再考虑。
- Lottie 文件应小而少，避免影响首屏性能。

## 10. 图表动效

### 10.1 首页 mini chart

- 默认静态或极轻入场。
- 不做动态绘线作为必要体验。

### 10.2 趋势详情图

- ECharts 动画默认关闭或缩短。
- 点击异常点后，批注卡点亮。
- 图表失败时，历史记录列表兜底。

## 11. reduced-motion 等价降级

微信小程序没有标准 Web `prefers-reduced-motion` 的完整等价时，可通过应用内设置或全局配置预留：

```text
motionLevel: normal | reduced | off
```

降级策略：

| 动效 | reduced | off |
|---|---|---|
| 档案卡浮入 | 仅 opacity 或直接显示 | 直接显示 |
| 指标批注点亮 | 静态高亮 | 静态标签 |
| 状态环展开 | 直接切换 | 直接切换 |
| 按钮点击反馈 | 保留轻 scale | 无缩放 |
| Lottie | 静态首帧 | 静态图标 |

## 12. 性能规则

- 不在长列表中对所有项目同时触发动效。
- 不在滚动过程中触发复杂动画。
- 不使用持续循环动画作为常驻元素。
- 首屏优先加载文字和卡片，再加载图表和动效。
- 低端设备优先保证滚动流畅。

## 13. 动效验收清单

- [ ] 三个微交互都有清晰使用场景。
- [ ] 动效只使用 transform / opacity 为主。
- [ ] 没有 `transition: all`。
- [ ] 异常指标不使用闪烁或大红呼吸灯。
- [ ] 状态环不做健康分和持续旋转。
- [ ] 所有关键信息在无动画状态下仍可理解。
- [ ] 有 reduced-motion 或等价降级方案。
- [ ] Lottie 不是首屏核心依赖。
