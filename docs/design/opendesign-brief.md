# OpenDesign 设计简报

用途：指导在 OpenDesign 中为 IeTecho 微信小程序绘制高保真视觉稿，并在完成后把参数回填到代码 token。

适用页面：首页、家人页、成员详情页、报告解读结果页（4 个核心页面）。

## 1. 一句话目标

画 4 个核心页面的高保真稿，让「家人档案卡」成为最强视觉记忆点，让中老年用户在 3 秒内看懂「谁需要关注、关注什么、下一步做什么」。

## 2. 画布与单位约束（必须遵守）

这些约束来自微信小程序运行环境，违反会导致还原困难。

| 项 | 值 | 原因 |
|---|---|---|
| 画布尺寸 | 375 × 812 px | iPhone 基准，小程序 rpx = px × 2 |
| 单位标注 | px（设计稿） | 还原时我按 rpx = px × 2 换算 |
| 顶部安全区 | 预留 88 px | 状态栏 + 导航栏 |
| 底部安全区 | 预留 34 px | Home indicator + 底部 Tab |
| 最小字号 | 13 px（仅标签） | 26 rpx，再小中老年看不清 |
| 正文字号 | 16 px | 32 rpx，中老年友好基准 |
| 关键数字 | ≥ 20 px | 突出指标值 |
| 最小点击区 | 44 × 44 px | 88 × 88 rpx，微信可访问性基线 |
| 页面左右边距 | 16 px | 32 rpx |
| 卡片间距 | 12-16 px | 工具界面，不做营销大留白 |

## 3. 交付物清单（按页面）

每个页面交付：1 张默认态 + 该页面关键状态（见下）。每张图标注关键尺寸。

### 3.1 首页（2 个状态）

```text
状态 A：默认态（有数据）
  ├─ 家庭健康总览（标题 + 「今天有 2 件事需要关注」）
  ├─ 今日重点卡（父亲｜血压连续 3 次偏高 + 建议 + 查看详情）
  ├─ 待办提醒（2 条，排序：逾期 > 今日 > 未来 7 天）
  ├─ 重点指标趋势（尿酸↑ 血糖→ LDL↑）
  ├─ 最近报告（2026-06-01 我的体检报告 + 查看解读）
  └─ 快捷操作（上传报告 / 新增提醒）

状态 B：首次使用空态
  ├─ 暖白引导卡「先建立你的健康档案」+ [上传我的体检报告] [完善我的基础信息]
  └─ 次卡「也可以管理家人健康」+ [添加家人]
```

首页最多 5 个模块。异常用柔和橙红，不用大面积红。

### 3.2 家人页（3 个状态）

```text
状态 A：首次使用（默认只有「我」，无档案）
  ├─ 「我」卡：还没有健康档案 + [上传我的体检报告] [完善基础信息]
  └─ 弱化卡：管理家人健康 + [添加家人]

状态 B：已有自己档案
  └─ 「我」卡：最近报告 + 重点关注 + 1 个复查提醒 + [查看档案]
     右上角小「+」添加家人（弱化）

状态 C：多家人
  └─ 家人档案卡列表（父亲/我/母亲），每卡：姓名 + 关系 + 状态标签 + 摘要
     排序：有提醒 > 有异常 > 最近更新 > 其他
```

家人档案卡是视觉记忆点，大圆角 22 px，轻阴影，关系色块头像。

### 3.3 成员详情页（1 个状态）

```text
默认态（父亲）：
  ├─ 顶部概览：父亲 + 67 岁 + 重点关注：血压血脂 + 2 项需关注｜1 个复诊提醒
  ├─ 健康状态环：2 项需关注（不显示健康分，只显示关注项数量）
  ├─ 最近报告：2026-06-01 体检报告 + 已解读｜待确认 2 个指标
  ├─ 重点指标趋势：血压↑ LDL↑ 血糖→
  ├─ 建议与提醒：06/20 心内科复诊 + 带近期血压记录
  └─ 历史记录：列表
```

健康状态环禁止健康分、排名、等级。用「N 项需关注」表达。

### 3.4 报告解读结果页（1 个状态）

```text
默认态：
  ├─ 报告解读 + 2026-06-01｜我的体检报告
  ├─ 报告摘要卡：本次报告有 3 项指标需要关注
  ├─ 异常指标卡（指标批注卡）：尿酸 468 μmol/L｜偏高 + 可能相关 + 建议
  ├─ 生活方式建议卡：饮食 + 运动
  ├─ 复查/就医提醒卡：1-3 个月后复查
  ├─ 免责声明（必须可见）
  └─ 底部主按钮：确认关键指标并保存
```

这一页是产品价值感最强的页面，重点打磨。

## 5. 视觉禁忌（不要画出来的东西）

- 禁止 AI 科技蓝紫渐变表达「智能」。
- 禁止大面积红色警报背景。异常用 `#c96f5f` 小面积。
- 禁止健康评分式状态环（「健康分 87」这种）。
- 禁止过度玻璃拟态 / 毛玻璃 / 复杂渐变叠加。小程序性能差。
- 禁止三张完全相同模板卡片堆叠。
- 禁止首页做营销 hero 大图。
- 禁止关键文字低对比度。
- 禁止混用多套图标风格。

## 6. 头像处理（不依赖真实头像）

MVP 用关系色块，不要花时间画真实头像插画。

```text
我：深绿
父亲：蓝灰
母亲：暖橙
伴侣：浅紫灰
孩子：浅黄绿
其他：中性灰蓝
```

## 7. 图表处理

首页和成员详情的趋势用简化 mini chart：折线或趋势箭头，不画复杂坐标轴。详情页趋势图留 P3 用 ECharts 实现，设计稿画占位即可。

颜色：

```text
正常趋势线：#4f7765
关注趋势线：#d9a35b
异常点：#c96f5f
参考范围底：#e7f1ec
```

## 8. 交付与回填流程

```text
1. 你在 OpenDesign 按 §3 画 4 页（共 7 个状态）
2. 每页导出 PNG，按 §4 标注关键尺寸
3. 贴给我，我对照 §4 token 表逐页回填
4. 我改 tokens.wxss + 各页面 wxss，WXML 和 TS 不动
5. 你在微信开发者工具看效果，迭代
```

逐页对齐，不一次性全做完。

## 9. 验收标准

每页画完自检：

- [ ] 375×812 画布，遵守 §2 约束。
- [ ] 家人档案卡是最强视觉记忆点。
- [ ] 3 秒能看出「谁需要关注、关注什么、下一步」。
- [ ] 异常表达克制（小面积橙红，不警报）。
- [ ] 字号、间距、点击区适合中老年。
- [ ] 不依赖大量插画或素材。
- [ ] 不出现 §5 任何禁忌项。
- [ ] 所有色值和数值来自 §4 token 表，无自创值。
