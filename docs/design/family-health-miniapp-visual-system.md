# 家庭健康小程序视觉系统

## 1. 视觉方向

```text
方向名称：温润健康管家（深绿体系）
关键词：温润、健康、可信、低压力、长期陪伴、中老年友好
视觉记忆点：家人档案卡
```

本视觉系统用于体检报告 / 化验单解读和家庭健康档案管理微信小程序。界面应降低用户面对异常指标时的焦虑，同时保持可信和清晰。

> 配色体系于 2026-06-18 经 OpenDesign 设计稿确认，由青绿（#35AFA0）调整为深绿（#4f7765），更沉稳自然。深绿体系以 `miniprogram/styles/tokens.wxss` 为唯一真源。

## 2. 设计原则

- 用暖白和低饱和青绿色建立健康感。
- 用大字号和明确层级照顾中老年阅读。
- 用柔和状态色表达异常，不做医疗恐吓。
- 用卡片组织家庭成员档案，但避免所有内容都变成相同卡片模板。
- 少素材依赖，主要靠排版、色彩、图表和轻动效形成质感。
- 首页是工具界面 (utility surface)，不做营销 hero。

| `--color-primary-deep` | `#36594c` | 深色文字、重要强调 |
| `--color-primary-soft` | `#e7f1ec` | 浅绿背景、轻状态底色 |
| `--color-primary-muted` | `#d5e2d5` | 图表参考区间、弱强调 |

### 3.2 页面和表面

| Token | 色值 | 用途 |
|---|---|---|
| `--color-page` | `#e7efec` | 页面冷灰绿背景 |
| `--color-surface` | `#fffdf8` | 主要卡片（暖白） |
| `--color-surface-soft` | `#e2eee8` | 次级卡片、输入背景 |
| `--color-surface-warm` | `#fffdf8` | 首次使用引导卡 |

### 3.3 文本

| Token | 色值 | 用途 |
|---|---|---|
| `--color-text-primary` | `#32443c` | 主文本 |
| `--color-text-secondary` | `#587066` | 次级文本 |
| `--color-text-muted` | `#8ea399` | 辅助说明 |
| `--color-text-inverse` | `#ffffff` | 深色按钮文本 |

### 3.4 状态色

| 状态 | Token | 色值 | 用途 |
|---|---|---|---|
| 平稳 / 正常 | `--color-status-normal` | `#4f7765` | 正常标签、趋势线 |
| 关注 / 提醒 | `--color-status-attention` | `#d9a35b` | 关注标签、今日提醒 |
| 异常 / 警示 | `--color-status-warning` | `#c96f5f` | 明确异常、逾期提醒 |
| 信息 / AI 解读 | `--color-status-info` | `#7aa6ad` | AI 解读状态 |
| 完成 | `--color-status-done` | `#789b67` | 已完成提醒 |

### 3.5 色彩边界

- 警示色只用于标签、点、短状态条，不做大面积背景。
- 不使用蓝紫渐变表达 AI。
- 不使用纯红表达健康异常。
- 文本颜色不能过浅；关键说明不能低于 `--color-text-secondary`。

## 4. 字体与字号

微信小程序中优先使用系统字体，避免依赖特殊字体资源。

建议字体栈：

```text
-apple-system, BlinkMacSystemFont, "PingFang SC", "Microsoft YaHei", sans-serif
```

### 4.1 字号 Token

| Token | 建议值 | 用途 |
|---|---:|---|
| `--font-page-title` | 24px | 页面标题 |
| `--font-section-title` | 20px | 模块标题 |
| `--font-card-title` | 18px | 卡片主标题 |
| `--font-body` | 16px | 正文、关键说明 |
| `--font-body-small` | 15px | 次正文 |
| `--font-caption` | 13px | 标签、辅助说明 |
| `--font-number-large` | 24px | 关键指标数字 |
| `--font-number` | 20px | 卡片指标数字 |

### 4.2 行高

| 类型 | 行高 |
|---|---:|
| 标题 | 1.25-1.35 |
| 正文 | 1.45-1.6 |
| 标签 | 1.2 |

### 4.3 中老年友好规则

- 关键正文不低于 15px。
- 主正文建议 16px。
- 重要数字不低于 20px。
- 主按钮文字不低于 16px。
- 不使用极细字重承载关键信息。

## 5. 间距系统

采用 4px 基础栅格。

```text
4 / 8 / 12 / 16 / 20 / 24 / 32 / 40
```

### 5.1 页面间距

| Token | 建议值 | 用途 |
|---|---:|---|
| `--space-page-x` | 16px | 页面左右边距 |
| `--space-section-y` | 20px | 模块间距 |
| `--space-card` | 16px | 卡片内边距 |
| `--space-card-large` | 20px | 大卡片内边距 |
| `--space-inline` | 8px | 标签、按钮内元素间距 |

### 5.2 规则

- 首页模块之间保持 12-16px，不做营销页式大留白。
- 卡片内主要信息间距 8-12px。
- 表单项之间至少 16px。
- 底部 Tab 需考虑 iPhone 安全区。

## 6. 圆角系统

定义固定圆角层级，避免随意取值。

| Token | 值 | 用途 |
|---|---:|---|
| `--radius-sm` | 8px | 小标签、内部元素 |
| `--radius-md` | 14px | 输入框、普通按钮 |
| `--radius-lg` | 18px | 小卡片 |
| `--radius-xl` | 22px | 家人档案卡、主卡片 |
| `--radius-pill` | 999px | 状态标签、胶囊按钮 |

规则：

- 家人档案卡使用 `--radius-xl`。
- 标签使用 `--radius-pill`。
- 嵌套圆角遵循内外层级，不让内层圆角大于外层。

## 7. 阴影与层级

### 7.1 阴影 Token

| Token | 建议值 | 用途 |
|---|---|---|
| `--shadow-card` | `0 4px 16px rgba(38, 50, 56, 0.08)` | 普通卡片 |
| `--shadow-card-soft` | `0 2px 10px rgba(38, 50, 56, 0.06)` | 次级卡片 |
| `--shadow-floating` | `0 8px 24px rgba(38, 50, 56, 0.12)` | 浮层 / 重点卡 |

### 7.2 规则

- 阴影用于层级，不用于装饰。
- 页面背景和卡片必须有明确层级差。
- 避免白卡片放在近白背景上只用极弱阴影，导致不可辨识。

## 8. 按钮系统

### 8.1 主按钮

```text
高度：44-48px
圆角：14-18px
字号：16px
背景：#4f7765
文字：#FFFFFF
```

用途：上传报告、开始解读、保存档案。

### 8.2 次按钮

```text
高度：44px
背景：#e7f1ec 或 #e2eee8
文字：#36594c 或 #32443c
```

用途：完善基础信息、先保存不解读、稍后查看。

### 8.3 文本按钮

- 点击区域仍需 ≥ 40×40。
- 不用过浅颜色。
- 用于查看全部、返回、编辑等低权重动作。

## 9. 状态标签系统

### 9.1 标签形态

```text
圆角：999px
高度：24-28px
字号：12-13px
左右内边距：8-10px
```

### 9.2 标签类型

| 标签 | 颜色建议 |
|---|---|
| 平稳 | 青绿浅底 + 深青文字 |
| 关注中 | 橙色浅底 + 深橙文字 |
| 偏高 / 偏低 | 橙色浅底 |
| 逾期 | 橙红浅底 |
| 待确认 | 蓝色浅底 |
| 解读中 | 蓝色浅底 |
| 已完成 | 绿色浅底 |

规则：标签短文案，不承载长解释。

## 10. 图标与图形

### 10.1 图标风格

- 圆角线性图标。
- 线宽统一。
- 不混用多套图标。
- 不大面积使用红十字。

### 10.2 家人成员头像

MVP 可用关系色块替代真实头像：

```text
我：深绿
父亲：蓝灰
母亲：暖橙
伴侣：浅紫灰
孩子：浅黄绿
其他：中性灰蓝
```

这些颜色只做轻辅助，不改变主视觉系统。

## 11. 图表系统

### 11.1 首页 mini chart

- 用简化折线或趋势箭头。
- 不展示复杂坐标轴。
- 最多展示 2-3 个重点指标。
- 颜色与指标状态一致。

### 11.2 趋势详情图

使用 ECharts for Weixin。

建议：

- 折线图展示时间趋势。
- 正常范围用浅色背景带。
- 异常点用小圆点标记。
- Tooltip 简洁：日期、值、参考范围、状态。
- 图表动画默认关闭或极轻量。

### 11.3 图表颜色

| 用途 | 色值 |
|---|---|
| 正常趋势线 | `#4f7765` |
| 关注趋势线 | `#d9a35b` |
| 异常点 | `#c96f5f` |
| 参考范围 | `#e7f1ec` |
| 网格线 | `rgba(38, 50, 56, 0.08)` |
| 坐标文字 | `#6B7C80` |

## 12. 空态与错误态视觉

### 12.1 空态

- 使用简洁几何占位或线性图标。
- 标题明确下一步。
- 说明文字短。
- 必须有主行动按钮。

示例：

```text
先上传一份体检报告
上传后可以解读报告，并持续追踪重点指标。
[上传我的体检报告]
```

### 12.2 错误态

- 不使用冷冰冰的错误码作为主文案。
- 说明发生了什么、用户能做什么。
- 保留重试路径。

示例：

```text
报告上传失败
已保留你填写的信息，可以重新上传。
[重试]
```

## 13. 可转 WXSS / SCSS Token 建议

后续实现可集中维护：

```text
styles/tokens.wxss
styles/theme.wxss
styles/global.wxss
```

Token 分类：

```text
--color-*
--font-*
--space-*
--radius-*
--shadow-*
--duration-*
--ease-*
```

原则：

- 页面不直接写散落色值。
- 状态色统一引用 token。
- 卡片、按钮、标签共享圆角和间距 token。

## 14. 视觉禁忌

- 禁止 AI 科技蓝紫渐变。
- 禁止大面积红色警报背景。
- 禁止健康评分式状态环。
- 禁止过度玻璃拟态。
- 禁止三张完全相同模板卡片堆叠而无信息层级。
- 禁止首页做营销 hero。
- 禁止关键文字低对比度。

## 15. 验收清单

- [ ] 首屏能立即看出这是家庭健康档案产品。
- [ ] 家人档案卡是最突出的视觉记忆点。
- [ ] 异常指标表达克制但明确。
- [ ] 字号、间距、点击区域适合中老年用户。
- [ ] 不依赖大量插画或素材。
- [ ] 所有状态色有明确用途。
- [ ] 图表服务理解，不做装饰。
- [ ] 视觉 token 能直接转成 WXSS / SCSS。
