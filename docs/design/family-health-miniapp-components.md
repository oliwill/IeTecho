# 家庭健康小程序核心组件规格

## 1. 组件设计原则

本组件规格服务于微信小程序页面原型和后续开发实现。组件语言必须符合「温润健康管家」方向：清晰、可信、低压力、中老年友好。

通用原则：

- 关键点击区域不小于 40×40。
- 关键信息字号不低于 15px。
- 不用强红色制造医疗恐吓。
- 卡片承载状态，不堆叠过多按钮。
- 所有 AI、指标、提醒组件都必须避免诊断式表达。
- 组件状态必须覆盖加载、空态、错误和处理中。

## 2. 组件优先级

| 组件 | MVP 优先级 | 说明 |
|---|---:|---|
| 家人档案卡 `FamilyProfileCard` | P0 | 视觉记忆点，首页、家人页、成员详情复用 |
| 报告解读卡 `ReportSummaryCard` | P0 | 报告上传后承接解读状态 |
| 指标批注卡 `MetricAnnotationCard` | P0 | 报告解读、指标确认、趋势异常点复用 |
| 提醒卡 `ReminderCard` | P0 | 首页和提醒管理核心组件 |
| 空态 / 错误态 `EmptyState` / `ErrorState` | P0 | 首次使用和失败兜底 |
| 状态标签 `StatusTag` | P0 | 指标、报告、提醒状态统一表达 |
| 底部 Tab `BottomTab` | P0 | 首页 / 家人 / 报告 / 我的 |
| 趋势图卡 `TrendChartCard` | P1 | Phase 3 指标趋势重点组件 |
| 健康状态环 `HealthStatusRing` | P1 | 成员详情聚合状态，不做健康评分 |
| 按钮体系 | P0 | 主按钮、次按钮、文本按钮 |

## 3. 家人档案卡 FamilyProfileCard

### 3.1 用途

家人档案卡是产品视觉记忆点，承担家庭成员健康状态入口。

使用位置：

- 家人页成员列表。
- 首页今日重点关联成员。
- 成员详情顶部概览。

### 3.2 字段

```text
memberId
name
relation
ageLabel
avatarType
focusTags[]
latestReportDate
status
statusText
nextAction
reminderCount
abnormalMetricCount
```

### 3.3 状态

| 状态 | 展示 |
|---|---|
| 无档案 | 引导上传报告 / 完善基础信息 |
| 平稳 | 青绿色状态标签，显示最近报告 |
| 关注中 | 橙色标签，显示需关注指标 |
| 待复查 | 显示复查日期和提醒入口 |
| 逾期提醒 | 柔和橙红标签，提供重新安排 |
| 加载中 | 卡片骨架屏 |
| 错误 | 显示加载失败和重试 |

### 3.4 结构

```text
┌────────────────────────┐
│ 头像 / 关系标识   状态标签 │
│ 姓名 / 关系              │
│ 最近报告：2026-06-01      │
│ 重点关注：尿酸、血脂       │
│ 下一步：3 天后复查         │
│ [查看档案]               │
└────────────────────────┘
```

### 3.5 视觉规则

- 大圆角：20-24px。
- 暖白或浅青背景。
- 轻阴影，不使用硬边框围栏感。
- 头像可先用关系色块或线性图标，不依赖真实头像。
- 卡片内只放一个主操作，避免拥挤。

### 3.6 交互

- 点击卡片进入成员详情。
- 新增成员后返回列表时，可触发「档案卡浮入」。
- 卡片浮入只使用 opacity + translateY。

## 4. 健康状态环 HealthStatusRing

### 4.1 用途

用于聚合成员当前需要关注的事项。

重要边界：不是健康评分，不显示“健康分 87”。

### 4.2 字段

```text
attentionCount
reminderCount
trendChangeCount
status
items[]
```

### 4.3 表达方式

```text
2 项需关注
1 个复查提醒
```

### 4.4 状态

| 状态 | 展示 |
|---|---|
| 平稳 | 青绿色环 + “近期平稳” |
| 关注中 | 橙色环 + 关注项数量 |
| 待复查 | 复查提醒优先展示 |
| 数据不足 | 灰蓝色环 + “数据还不够形成趋势” |
| 加载中 | 静态骨架 |

### 4.5 交互

- 点击展开关注项明细。
- 展开动效是第三优先级微交互。
- 不自动在首页首屏展开。

### 4.6 禁止

- 禁止健康分、排名、等级化表达。
- 禁止持续旋转环。
- 禁止用红色大面积填充。

## 5. 报告解读卡 ReportSummaryCard

### 5.1 用途

展示一份报告的状态和下一步操作。

使用位置：

- 首页最近报告。
- 成员详情最近报告。
- 报告列表。

### 5.2 字段

```text
reportId
memberName
reportType
reportDate
hospital
fileType
interpretationStatus
summary
pendingMetricCount
primaryAction
```

### 5.3 状态

| 状态 | 主操作 |
|---|---|
| 未解读 | 开始解读 |
| 解读中 | 稍后查看 |
| 已解读 | 查看解读 |
| 解读失败 | 重新解读 / 手动录入 |
| 待确认指标 | 确认指标 |
| 已保存档案 | 查看档案 |

### 5.4 结构

```text
┌────────────────────────┐
│ 体检报告        已解读    │
│ 2026-06-01｜瑞金医院      │
│ 本次报告有 3 项指标需关注  │
│ 待确认 2 个指标           │
│ [查看解读]               │
└────────────────────────┘
```

### 5.5 规则

- 日期和医院是次级信息。
- 解读状态使用轻标签。
- 解读失败时不暗示报告丢失。

## 6. 指标批注卡 MetricAnnotationCard

### 6.1 用途

解释单个指标的当前值、参考范围、异常状态和下一步建议。

使用位置：

- 报告解读结果页。
- 指标确认页。
- 趋势详情页异常点。

### 6.2 字段

```text
metricId
metricName
value
unit
referenceRange
status
category
plainLanguageExplanation
possibleFactors[]
nextSuggestion
confirmationState
```

### 6.3 状态

| 状态 | 表现 |
|---|---|
| 正常 | 青绿色 / 中性标签 |
| 偏高 | 柔和橙色标签 |
| 偏低 | 柔和橙色标签 |
| 明确异常 | 小面积橙红标签 |
| 待确认 | 待确认标签 + 编辑入口 |
| 已修改 | 已修改轻标签 |
| 识别不确定 | 提示用户核对 |
| 已忽略 | 折叠或弱化显示 |

### 6.4 结构

```text
┌────────────────────────┐
│ 尿酸              偏高    │
│ 468 μmol/L               │
│ 参考范围：149-416         │
│ 可能与高嘌呤饮食、饮酒、代谢因素相关 │
│ 建议：减少酒精和高嘌呤食物，按计划复查 │
│ [编辑] [忽略]             │
└────────────────────────┘
```

### 6.5 交互

- 点击展开通俗解释。
- 指标确认页可编辑数值、单位、参考范围。
- 趋势图点击异常点时，对应批注卡点亮。

### 6.6 动效

「指标批注点亮」只做一次轻量高亮：

- opacity / transform / 背景色轻过渡。
- 不闪烁，不循环。
- 同屏最多 3 个卡片错峰点亮。

## 7. 提醒卡 ReminderCard

### 7.1 用途

展示复查、复诊、生活习惯提醒。

使用位置：

- 首页待办提醒。
- 成员详情建议与提醒。
- 提醒列表。

### 7.2 字段

```text
reminderId
memberName
type
title
time
repeatRule
note
status
relatedMetric
```

### 7.3 状态

| 状态 | 表现 |
|---|---|
| 今日 | 置顶，清晰时间 |
| 未来 | 中性显示 |
| 逾期 | 柔和橙红标签，提供改期 |
| 已完成 | 弱化，可折叠 |
| 已取消 | 不在默认列表显示 |
| 订阅未授权 | 显示小提示，不阻塞 |

### 7.4 结构

```text
┌────────────────────────┐
│ 父亲｜心内科复诊          │
│ 06/20 09:00             │
│ 带近期血压记录            │
│ [完成] [改期]            │
└────────────────────────┘
```

### 7.5 规则

- 首页只展示 1-2 条最重要提醒。
- 逾期提醒不使用大面积红底。
- 不承诺后台强推。

## 8. 趋势图卡 TrendChartCard

### 8.1 用途

展示指标长期变化摘要。

使用位置：

- 首页重点指标趋势。
- 成员详情趋势摘要。
- 指标趋势详情页。

### 8.2 字段

```text
metricName
currentValue
unit
trendDirection
status
lastUpdated
points[]
referenceRange
```

### 8.3 状态

| 状态 | 表现 |
|---|---|
| 数据不足 | 提示需要至少 2 条记录 |
| 稳定 | 青绿色趋势 |
| 上升需关注 | 橙色趋势标签 |
| 下降需关注 | 橙色趋势标签 |
| 异常点 | 小圆点标记 |
| 图表失败 | 历史记录列表兜底 |

### 8.4 图表规则

- 首页用 mini chart，不展示复杂坐标。
- 详情页用完整 ECharts。
- 正常范围用浅色带。
- 数字使用等宽数字样式思路，保证纵向对齐。

## 9. 底部 Tab BottomTab

### 9.1 内容

```text
首页 / 家人 / 报告 / 我的
```

> 2026-06-18 经 OpenDesign 设计稿确认，由 3 Tab（首页/家人/更多）调整为 4 Tab（首页/家人/报告/我的），新增「报告」作为跨家人报告列表的一级入口，「更多」改名为「我的」。

### 9.2 状态

- 默认态。
- 当前选中态。
- 有提醒轻提示态。
- reduced-motion 静态态。

### 9.3 规则

- 点击区域至少 40×40。
- 选中态使用深绿 `#36594c`。
- 不使用复杂弹跳动画。
- 不新增 AI 一级 Tab。

## 10. 空态 EmptyState

### 10.1 用途

首访、无报告、无指标、无提醒、数据不足。

### 10.2 结构

```text
┌────────────────────────┐
│ 简洁图标 / 几何占位       │
│ 标题：先上传一份体检报告   │
│ 说明：上传后可以解读报告并持续追踪指标 │
│ [上传我的体检报告]        │
└────────────────────────┘
```

### 10.3 规则

- 不只写“暂无数据”。
- 必须给下一步行动。
- 不依赖大插画。

## 11. 错误态 ErrorState

### 11.1 用途

网络错误、上传失败、解读失败、保存失败。

### 11.2 规则

- 文案说明具体失败点。
- 提供重试或返回路径。
- 保留用户已填写内容。
- 不使用“Oops”。

示例：

```text
报告解读没有完成，原始报告已保存。你可以稍后重试，或先手动录入关键指标。
```

## 12. 按钮体系

### 12.1 主按钮

用途：页面主行动，如上传报告、开始解读、保存档案。

规则：

- 高度不低于 44px。
- 字号不低于 16px。
- 主色使用青绿色。
- active 状态可轻微 scale(0.96)。

### 12.2 次按钮

用途：补充操作，如完善基础信息、先保存不解读。

规则：

- 使用浅色背景或弱边界。
- 不与主按钮抢视觉。

### 12.3 文本按钮

用途：查看全部、稍后查看、返回。

规则：

- 点击区域仍需至少 40×40。
- 不用过浅颜色。

## 13. 状态标签 StatusTag

### 13.1 类型

```text
平稳
关注中
偏高
偏低
待复查
逾期
待确认
已修改
解读中
解读失败
```

### 13.2 规则

- 胶囊形状。
- 小面积使用状态色。
- 不用状态色承载长段文字。
- 警示色避免大面积填充。

## 14. 组件验收清单

- [ ] 家人档案卡是最强记忆点。
- [ ] 健康状态环不做健康分。
- [ ] 指标批注卡能用于解读、确认和趋势。
- [ ] 报告解读卡覆盖未解读、解读中、失败、待确认。
- [ ] 提醒卡覆盖今日、未来、逾期、已完成。
- [ ] 空态都有下一步行动。
- [ ] 错误态保留用户上下文。
- [ ] 所有交互区域不小于 40×40。
