## 一、编译能否通过(必过)

- [ ] 编译结果 0 error。有 error 先解决,后面全部不用看。

## 三、深绿体系验证(本次核心变更)

对照这些位置,确认是深绿不是青绿:

- [ ] 首页标题区背景偏冷灰绿(#e7efec),不是暖白(#f8f6f1)。
- [ ] 卡片底色偏暖白(#fffdf8)。
- [ ] 主按钮(如有)是深绿 #4f7765,不是青绿 #35afa0。
- [ ] 状态标签正常态是深绿。

> 如果某处仍是青绿,说明该页面 wxss 写死了旧色值,把页面名告诉我,我去改。

## 七、设计稿质感核对(诚实预期)

设计稿 home.html 的这些效果,我在小程序里**有取舍**,真机核对实际表现:

| 设计稿效果 | 小程序实际 | 你需要判断 |
|---|---|---|
| 纸感颗粒纹理 | 已移除(性能) | 是否觉得太平,需要补? |
| 毛玻璃卡片 | 已改成实色卡片 | 是否觉得不够精致? |
| oklch 半透明叠加 | 已改成 hex 实色 | 深绿是否偏沉闷? |
| 启动页剪纸插画 | 暂未还原到小程序 | 是否要补启动页? |

这四项是已知的质感损失。如果真机上你觉得整体太朴素,我们再针对性补 Lottie 或轻量纹理,但要权衡性能。
